LibRapid: A High-Performance Multidimensional Array Library

Optimisation

LibRapid is an open-source library for high-efficiency calculations, written primarily in/for C++, though with a fully-featured Python interface, a Rust port and plans for other languages.

Why is Speed Important?

When writing a program for a hobby project or simple application, performance generally isn't a huge concern to developers. It makes no difference if the program executes in \(5ms\) or \(10ms\), and hence programmers will often choose the easiest solution, as opposed to the most efficient solution. In certain applications, however, the efficiency and performance of a program is hugely important.

Applications where Performance is Critical

The requirements for performance in different applications varies, however the general reasoning behind the need for fast code is generally the same. Firstly, if a program needs to be responsive and fluid (e.g. a video game), the underlying program must update quickly enough to accept user input and output the response with minimal latency. This requires the program to be efficiently implemented, as slow code will reduce the overall experience of the application.

The second common reason for high-performance code is when the code needs to be run millions of times to achieve its goal (e.g. a neural network). In this case, optimising a single function to save a few microseconds can potentially save hours when the final program is run.

It's cases like these, where performance is critical to the application serving it's purpose, that high-performance code is needed. Without efficient, low-level implementations, the resulting program may simply be too slow to be practical to use.

What Optimisations are Present in LibRapid?

LibRapid is written in C++, which already makes it faster than many other languages such as Python, JavaScript and Java, but there are a huge number of other optimisations in place to further accelerate the library.

1

Data Organisation

The aligned, contiguous storage of the Array class allows for more data prefetching, more cache hits and better optimisation by the compiler.

2

Lazy-Evaluated Results

LibRapid avoids calculating the result of an operation as far as possible, only evaluating the result when it is absolutely required.

3

Multithreaded Code

By running large loops in parallel, code execution can be dramatically improved. Other optimisations include forcing single-threaded code for small matrices, as the overhead of spawning threads can reduce performance.

4

Vectorised Instructions

By using SIMD machine-code instructions present in your CPU, LibRapid can perform 2, 4, 8 or even 16 calculations per instruction call, resulting in improved cache hits and more efficient processing.

5

Instruction Concatenation

Operations on Arrays are concatenated together, forming a single, optimised loop at compile time. This means you can string more operations together without dramatically increasing performance.

Basic Example

// Create an array of floats with dimensions [234, 567]
auto x = lrc::Array<float>(lrc::Extent(234, 567));

// Fill the array with random values
for (int i = 0; i < 234; ++i) {
    for (int j = 0; j < 567; ++j) {
        x[i][j] = lrc::random(-1, 1);
    }
}

// Print the array, formatting values to 3dp
fmt::print("{:.3f}\n", x);

Why use LibRapid

  • Faster execution than competing libraries
  • Support for CPU and GPU arrays, as well as extended data types, such as float16
  • Automatic packaging and linking with BLAS libraries
  • Simple, consistent, easy to use interface for C++, Rust and Python
  • Wide variety of helper functions and classes to ease development
  • Automatic detection of CPU/GPU capabilities and corresponding optimisations

Why not to use LibRapid

  • Fewer functions than more well-established libraries
  • Less documentation and fewer tests than existing libraries
  • Higher potential for bugs than in larger libraries
